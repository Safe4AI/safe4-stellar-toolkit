Paid AI tools on Stellar

Safe4 Stellar Toolkit

This demo shows a simple pattern: a tool call does not execute just because a payment exists. The server issues a payment requirement, checks proof against the original request, applies policy, and only then returns the tool result with a receipt.

The demo keeps two proof modes visible: a reliable mock flow for live demos and a real Stellar transaction-hash path for a stronger submission claim.

How the flow works

  • Request a paid tool and receive a 402 payment challenge.
  • Settle it with mock proof or a real Stellar testnet payment.
  • Retry with the original X-Request-Id and payment proof.
  • Receive an authorized result, receipt, and audit entry.
