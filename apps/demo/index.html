Control every paid tool call before it executes.

Safe4 sits between an AI agent and a monetised tool. The server issues a payment challenge, verifies payment context, applies local policy, surfaces external risk, and only then releases the result with a receipt and audit trail.

This demo shows the shape a real agent payment control plane should have on Stellar: payment-aware, policy-aware, and receipt-backed by default.

Request binding

Payment is tied to a specific request.

The server binds the tool name, payload, memo, amount, and request ID together so a payment proof cannot be replayed onto a different call.

Policy management

Policy is visible, not hidden.

Spend caps, rate limits, tool denies, and external risk recommendations are exposed through explicit endpoints so teams can inspect what the system would do before they trust it.

Receipts and audit

Execution returns evidence.

Authorized results include payment references, policy decisions, timestamps, and audit records so the tool call is defensible after the fact.
